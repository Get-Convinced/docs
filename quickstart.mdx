This walkthrough takes you from "I have a partner API key" to "the visitor sees a widget grounded in their account intel" using **auto-enrich mode** — the lowest-friction path. You hand Convinced rep notes and a few links; the AI research pipeline synthesizes the structured intel for you.

<Tip>
  If you already have structured account intel from your CRM, jump to [partner-intel mode](/guides/campaigns/partner-intel) — it's synchronous and the widget is ready as soon as the request returns.
</Tip>

## Prerequisites

- A Convinced organization. If you don't have one yet, contact [hi@getconvinced.ai](mailto:hi@getconvinced.ai).
- A partner API key (we'll create one in step 1).
- A terminal with `curl` and `jq`.

## The 5-minute path

<Steps>
  <Step title="Create a partner API key">
    In the [dashboard](https://app.getconvinced.ai), open **Admin → API keys** under your org:

    ```
    https://app.getconvinced.ai/org/<your-org-slug>/admin/api-keys
    ```

    Click **New key**, name it (for example `partner-outbound-pipeline`), and copy the key once — it's only shown to you at creation time.

    <Warning>
      API keys are organization-bound. Never share a key across organizations, and never commit one to a public repo or expose it in client-side code.
    </Warning>
  </Step>

  <Step title="Set environment variables">
    Set the API base and your key in your shell. The base host is the KB MCP service (not the dashboard URL).

    <CodeGroup>
    ```bash bash
    export CONVINCED_API_BASE="http://convinced-kb-mcp.railway.internal"
    export CONVINCED_API_KEY="<your-partner-api-key>"
    ```

    ```fish fish
    set -x CONVINCED_API_BASE "http://convinced-kb-mcp.railway.internal"
    set -x CONVINCED_API_KEY "<your-partner-api-key>"
    ```
    </CodeGroup>

    Every request sends `Authorization: Bearer $CONVINCED_API_KEY`.
  </Step>

  <Step title="Create a campaign">
    Campaigns group target accounts under a stable `slug`. Calling this endpoint again with the same slug updates the campaign's name or description; the slug itself is immutable.

    <CodeGroup>
    ```bash curl
    curl -X POST "$CONVINCED_API_BASE/api/campaigns" \
      -H "Authorization: Bearer $CONVINCED_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "name": "Q2 Logistics Outreach",
        "slug": "q2-logistics-outreach",
        "description": "Outbound campaign for logistics accounts"
      }'
    ```

    ```json response
    {
      "campaign": {
        "id": "11111111-1111-1111-1111-111111111111",
        "name": "Q2 Logistics Outreach",
        "slug": "q2-logistics-outreach",
        "description": "Outbound campaign for logistics accounts",
        "status": "active",
        "createdAt": "2026-05-04T10:00:00.000Z",
        "updatedAt": "2026-05-04T10:00:00.000Z"
      }
    }
    ```
    </CodeGroup>

    <Note>
      Only campaigns with `status: "active"` resolve targets at the widget. Campaigns are created active by default.
    </Note>
  </Step>

  <Step title="Push a target with auto-enrich">
    Send rep notes (`rawResearch`, plain text or markdown), optional landing-page URLs (`links`), and `enrich: true`. Convinced runs web research, KB matching, and LLM synthesis in the background and writes a structured `intel` object back to the row. The `token` becomes the `?c=<token>` query string for the widget.

    <CodeGroup>
    ```bash curl
    curl -X POST "$CONVINCED_API_BASE/api/campaigns/q2-logistics-outreach/targets" \
      -H "Authorization: Bearer $CONVINCED_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "companyName": "Acme Logistics",
        "token": "acme-logistics",
        "enrich": true,
        "rawResearch": "## Account notes\n- Mid-market 3PL, last-mile dispatch for grocery + pharmacy.\n- ~300 drivers, expanding into Tier-2.\n- Champion: Priya Rao, VP Ops. Cares about dispatch SLA + fleet utilization.\n- Currently on SAP TMS + Locus, evaluating FarEye for visibility.",
        "links": [
          "https://acmelogistics.example.com/about",
          "https://acmelogistics.example.com/news/series-b"
        ],
        "additionalContext": "Lead with ROI on dispatch speed; this account misses peak-window SLAs."
      }'
    ```

    ```json response
    {
      "target": {
        "id": "22222222-2222-2222-2222-222222222222",
        "campaignId": "11111111-1111-1111-1111-111111111111",
        "companyName": "Acme Logistics",
        "token": "acme-logistics",
        "enrichedAt": null,
        "enrichmentStatus": "pending",
        "createdAt": "2026-05-04T10:00:01.000Z",
        "updatedAt": "2026-05-04T10:00:01.000Z"
      },
      "targetId": "22222222-2222-2222-2222-222222222222",
      "widgetToken": "acme-logistics",
      "widgetParam": "c",
      "widgetQuery": "?c=acme-logistics"
    }
    ```
    </CodeGroup>

    <Tip>
      `token` is optional — if you omit it, Convinced derives one from `companyName` (lowercase, hyphenated, max 64 chars). Setting it explicitly is recommended so your outbound links stay stable.
    </Tip>
  </Step>

  <Step title="Poll until enrichment is done">
    The auto-enrich pipeline runs asynchronously. Poll `GET /api/campaigns/{slug}/targets/{token}` until `enrichmentStatus` flips to `done`. Typical end-to-end time is 30–90 seconds.

    <CodeGroup>
    ```bash one-shot
    curl "$CONVINCED_API_BASE/api/campaigns/q2-logistics-outreach/targets/acme-logistics" \
      -H "Authorization: Bearer $CONVINCED_API_KEY"
    ```

    ```bash polling-loop
    # Polls every 5 seconds for up to 3 minutes.
    until status=$(curl -sS \
      "$CONVINCED_API_BASE/api/campaigns/q2-logistics-outreach/targets/acme-logistics" \
      -H "Authorization: Bearer $CONVINCED_API_KEY" | jq -r '.target.enrichmentStatus') \
      && [[ "$status" == "done" || "$status" == "failed" || "$status" == "skipped" ]]; do
      echo "status=$status — waiting…"
      sleep 5
    done
    echo "final status: $status"
    ```

    ```json response
    {
      "target": {
        "id": "22222222-2222-2222-2222-222222222222",
        "campaignId": "11111111-1111-1111-1111-111111111111",
        "campaignName": "Q2 Logistics Outreach",
        "campaignSlug": "q2-logistics-outreach",
        "campaignStatus": "active",
        "companyName": "Acme Logistics",
        "token": "acme-logistics",
        "enrichedAt": "2026-05-04T10:01:14.000Z",
        "enrichmentStatus": "done",
        "enrichmentError": null,
        "enrichmentStartedAt": "2026-05-04T10:00:05.000Z",
        "hasIntel": true,
        "widgetReady": true,
        "updatedAt": "2026-05-04T10:01:14.000Z"
      },
      "targetId": "22222222-2222-2222-2222-222222222222",
      "widgetToken": "acme-logistics",
      "widgetParam": "c",
      "widgetQuery": "?c=acme-logistics"
    }
    ```
    </CodeGroup>

    <Note>
      `enrichmentStatus` lifecycle: `pending` → `running` → `done` (or `failed`, with `enrichmentError` populated). When it's `done` and `widgetReady: true`, send the visitor in.
    </Note>
  </Step>

  <Step title="Open the widget">
    The visitor needs to land on a page where the widget can resolve the campaign token. Pick whichever option fits your channel — your `<org-slug>` is the slug shown in the dashboard URL (e.g. `/org/convinced/...` → `convinced`).

    <Tabs>
      <Tab title="Embed + URL token">
        Drop the loader on your own page and send visitors to a URL with `?c=<token>`. Best when you already have a marketing page and want to A/B test landings.

        ```html
        <script
          src="https://app.getconvinced.ai/widget/loader.js"
          data-org="<org-slug>"
          async
        ></script>
        ```

        Outbound link: `https://yourcompany.com/demo?c=acme-logistics`
      </Tab>

      <Tab title="Embed + hard-pinned token">
        Use when one page is dedicated to a single account.

        ```html
        <script
          src="https://app.getconvinced.ai/widget/loader.js"
          data-org="<org-slug>"
          data-c="acme-logistics"
          async
        ></script>
        ```
      </Tab>

      <Tab title="Pretty URL">
        The loader picks up the token from the path segment automatically — clean for email and SMS.

        ```
        https://yourcompany.com/for/acme-logistics/
        ```
      </Tab>

      <Tab title="Hosted full-page">
        No embed needed. Send the visitor straight to Convinced — best for cold outbound where you don't have a landing page yet.

        ```
        https://app.getconvinced.ai/widget/v2/<org-slug>?c=acme-logistics
        ```
      </Tab>
    </Tabs>

    Once the visitor opens the link, the widget pulls the intel for `acme-logistics` and the agent runs the session grounded in the account's pain points, champion, current stack, and competitor situation.
  </Step>
</Steps>

## What happens behind the scenes

When you call the auto-enrich endpoint, Convinced:

1. **Crawls** the URLs in `links` and pulls clean text.
2. **Researches** the company on the public web for industry, size, and recent signals.
3. **Matches** the account to your knowledge base — referenceable customers, competitor data, product modules.
4. **Synthesizes** a structured `intel` object via LLM, validating it against the schema.
5. **Writes** the intel back to the target row and flips `enrichmentStatus` to `done`.

Read [auto-enrich mode](/guides/campaigns/auto-enrich) for the full pipeline and how to write `rawResearch` that produces good intel.
